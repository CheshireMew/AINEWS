# 去重算法详解：从入门到精通

## 🎯 算法目标

找出所有相似的新闻，让后来的新闻指向先发布的新闻，避免链式结构。

---

## 📊 第一步：计算相似度

### 对于每条新闻：
1. **提取关键词**
   - 把标题分词（如"比特币价格上涨" → ["比特币", "价格", "上涨"]）
   - 去除停用词（如"的"、"了"等无意义词）
   - 提取数字（如"10万美元" → 100000）

2. **发现动态停用词**
   - 如果"比特币"出现在50%以上的新闻中 → 太常见 → 从关键词中删除
   - 这样可以提高区分度

3. **两两对比**
   - 对于1000条新闻，要比较 1000×999÷2 = 499,500 次
   - 每次比较：
     - 关键词匹配度（50%权重）
     - 字符串相似度（20%权重）
     - 数字匹配度（30%权重）
   - 如果相似度 ≥ 0.50 → 标记为"重复对"

---

## 📝 第二步：标记重复

拿到所有"重复对"后（比如找到100对），开始标记：

### 规则：
- **时间早的** = Master（原始新闻）
- **时间晚的** = Duplicate（重复新闻）

### 示例：
```
新闻A (10:00) 和 新闻B (10:05) 相似
→ B指向A (B.duplicate_of = A.id)
```

---

## 🔧 第三步：防止链式结构

### 使用两个集合追踪：

1. **already_duplicate** = 已经被标记为duplicate的新闻
   - 作用：一个新闻只能被标记一次
   - 防止：A→B之后，又有A→C

2. **used_as_master** = 已经作为master的新闻
   - 作用：防止master后来又被标记为duplicate
   - 防止：C→B之后，又有B→A（形成链式C→B→A）

---

## 🆕 第四步：Master合并（新功能）

### 场景：
```
初始状态：
- 新闻C → 新闻B  (C指向B，B是master)
- 新闻D → 新闻B  (D也指向B)

后来发现：
- 新闻A 和 新闻B 相似！(A更早)
```

### 问题：
按旧逻辑，B已经在`used_as_master`中，不能被标记为duplicate，所以跳过。
结果：**A和B虽然相似，但无法合并**！

### 新的Master合并逻辑（已优化）：

**当发现duplicate在`used_as_master`中时：**
1. **关键验证**：首先计算 `other_news` 与 `new_master` 的相似度。
2. **只有相似度 ≥ 阈值 (0.50) 时**：
   - 找出所有指向B的新闻（C和D）
   - 把它们改为指向A
   - C.duplicate_of = A
   - D.duplicate_of = A
   - 理由更新为：`master合并,相似度xx`
3. **如果相似度不足**：
   - 保持原状，不进行合并
   - 防止因A和B相似（高），但A和C不相似（低）导致的错误合并。

**最终结果（仅当验证通过）：**
```
C → A
D → A  
B → A
```

所有新闻都合并到最早的master（A）！通过相似度二次验证，避免了"一锅端"导致的误判。

---

## ❓ 为什么ID 1223和1241没有合并？

### 可能原因：

1. **它们不在同一批去重中**
   - 如果1223在第一次去重，1241在第二次去重后才抓取
   - 第二次去重时1223不在news_list中
   - 所以无法发现它们相似

2. **时间窗口限制**
   - 虽然您说用了"全部时间"
   - 但代码中`find_duplicates`可能有其他过滤逻辑

3. **顺序问题**
   - 如果它们都在列表中，但已经各自有duplicates
   - Master合并应该会触发
   - 如果没触发，可能是代码bug

## 🔍 关键代码解析

### 核心检查逻辑：
```python
# 两个关键检查
if duplicate_id in already_duplicate or master_id in already_duplicate:
    continue  # 跳过，不标记

# Master合并检查
if duplicate_id in used_as_master:
    # 找出所有指向duplicate的新闻
    for other_news in news_list:
        if other_news.get('duplicate_of') == duplicate_id:
            other_news['duplicate_of'] = master_id  # 改指向
```
