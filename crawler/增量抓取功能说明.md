# 增量抓取功能说明

## 功能概述

增量抓取功能可以避免重复抓取已有的新闻，只抓取新增的内容。

## 工作原理

### 1. 加载历史记录
抓取前，从数据库读取该网站最新的一条新闻：
```python
scraper.load_last_news(db)
```

### 2. 抓取时定位
在抓取新闻列表时，逐条检查：
- 如果标题或URL匹配上次的新闻，停止抓取
- 否则继续抓取

### 3. 匹配策略
两种匹配方式（任一匹配即停止）：
- **标题匹配**：去除空格后完全相同
- **URL匹配**：URL完全相同

### 4. 降级方案
如果无法定位（新闻列表已刷新），会抓取所有能抓到的新闻。

## 代码示例

### 在爬虫中使用
```python
from database.db import Database
from scrapers.techflow import TechFlowScraper

db = Database()
db.connect()

scraper = TechFlowScraper()

# 启用增量抓取（默认启用）
scraper.load_last_news(db)

# 抓取（只会抓到新增的）
news_list = await scraper.run()

# 保存到数据库
for news in news_list:
    db.insert_news({...})

db.close()
```

### 禁用增量抓取
```python
scraper = TechFlowScraper()
scraper.incremental_mode = False  # 禁用增量抓取
news_list = await scraper.run()   # 抓取所有
```

## 实现细节

### BaseScraper基类
```python
class BaseScraper:
    def __init__(self, site_name, base_url):
        self.last_news_title = None
        self.last_news_url = None
        self.incremental_mode = True
    
    def load_last_news(self, db):
        """从数据库加载最新新闻"""
        latest = db.get_latest_news(self.site_name)
        if latest:
            self.last_news_title = latest['title']
            self.last_news_url = latest['source_url']
    
    def should_stop_scraping(self, title, url):
        """判断是否应该停止抓取"""
        if title == self.last_news_title:
            return True
        if url == self.last_news_url:
            return True
        return False
```

### 在具体爬虫中使用
```python
class TechFlowScraper(BaseScraper):
    async def scrape_important_news(self):

```

## 优点

1. **节省资源**：避免重复抓取和处理
2. **提高效率**：只处理新增内容
3. **降低负载**：减少详情页访问次数
4. **自动容错**：定位失败时自动降级为全量抓取

## 注意事项

1. **首次运行**：数据库为空时，会抓取所有新闻
2. **新闻顺序**：假设新闻按时间倒序排列（最新在前）
3. **标题变化**：如果网站修改了历史新闻标题，可能无法定位
4. **刷新过快**：如果网站刷新太快，上次新闻已不在列表，会抓取全部
