## ⚙️ 配置文件

位置: `config/scrape_intervals.yaml`

```yaml
# 爬取频率配置（单位：分钟）
# 建议：根据媒体更新频率合理设置，避免过度抓取

scrape_intervals:
  # 高频快讯类（推荐：15-60分钟）
  blockbeats: 30      # BlockBeats - 极高频更新
  
  # 中频综合类（推荐：60-90分钟）
  techflow: 60        # TechFlow深潮 - 快讯+深度
  foresight: 60       # Foresight News - 综合资讯
  marsbit: 60         # MarsBit - 综合资讯
  chaincatcher: 60    # ChainCatcher - 快讯+深度
  panews: 60          # PANews - 综合资讯
  
  # 低频深度类（推荐：90-180分钟）
  odaily: 90          # Odaily - 深度报道为主
  followin: 120       # Followin - 聚合平台

# 全局设置
global:
  # 默认频率（如果某个媒体未单独配置）
  default_interval: 60
  
  # 最小间隔限制（防止过度频繁）
  min_interval: 15
  
  # 重试设置
  retry_on_error: true
  retry_interval: 5   # 出错后等待5分钟重试
  max_retries: 3

# 时段调整（可选 - 未来功能）
# 不同时段可以使用不同的抓取频率
time_based:
  enabled: false
  # 工作日白天（活跃时段）- 更频繁
  weekday_daytime:
    hours: "09:00-18:00"
    multiplier: 0.8   # 频率提高20%（间隔缩短20%）
  
  # 夜间和周末 - 降低频率
  nighttime:
    hours: "00:00-06:00"
    multiplier: 2.0   # 频率降低50%（间隔翻倍）
```

## 🔧 使用方法

### 1. 查看当前配置
```python
from config_loader import load_scrape_config

config = load_scrape_config()
print(config['scrape_intervals'])
```

### 2. 手动调整
直接编辑 `config/scrape_intervals.yaml` 文件，修改对应媒体的数值。

### 3. 应用到定时任务
```python
from apscheduler.schedulers.background import BackgroundScheduler

scheduler = BackgroundScheduler()

# 为每个媒体设置不同的抓取频率
for site, interval_minutes in config['scrape_intervals'].items():
    scheduler.add_job(
        scrape_site,
        'interval',
        minutes=interval_minutes,
        args=[site],
        id=f'scrape_{site}'
    )

scheduler.start()
```

## 💡 调整建议

### 场景1: 资源有限（VPS性能较低）
```yaml
# 降低所有频率
blockbeats: 60      # 30 → 60
techflow: 90        # 60 → 90
其他: 120           # 60 → 120
```

### 场景2: 追求时效性（资源充足）
```yaml
# 提高关键媒体频率
blockbeats: 15      # 30 → 15
techflow: 30        # 60 → 30
其他: 45            # 60 → 45
```

### 场景3: 节省API成本
```yaml
# 适当降低频率，配合本地去重
blockbeats: 45
techflow: 90
其他: 120
```
