# 本地去重功能 - 数字容差匹配优化

## ✅ 已解决的问题

**用户反馈**：
> "4.6 万和46,379枚是一个很接近的数字，媒体会用不同单位表示（万、千、百万、亿等），但说的是同一个数字。同一时间段的新闻相似度够高就可以判定为重复。"

**解决方案**：添加数字容差匹配功能

---

## 🎯 核心改进

### 1. 数字近似匹配（±5%容差）
```python
46,000 vs 46,379 → 相差 0.8% → ✅ 认为相同
500万 vs 480万 → 相差 4% → ✅ 认为相同  
3000 vs 4000 → 相差 25% → ❌ 不同
```

**算法**：
```python
相对误差 = |数字1 - 数字2| / max(数字1, 数字2)
如果相对误差 ≤ 5%，认为是相同数字
```

### 2. 权重优化
```
旧版本:
- 字符串序列: 20%
- 数字匹配: 30%
- 关键词匹配: 50%
- 数字奖励: +0.4

新版本:
- 字符串序列: 15% ↓
- 数字匹配: 25% ↓  
- 关键词匹配: 60% ↑
- 数字奖励: +0.5 ↑
```

### 3. 更激进的去重策略
- **默认阈值**: 0.65（之前0.7）
- **数字权重**: 更重视数字和关键词，降低字符串相似度要求
- **适合多媒体源**: 不同媒体对同一事件的报道方式多样
